### Lab 4: Conditional VAE for Video Prediction

📌任務目標: 透過 Conditional VAE (CVAE) 實現影片序列預測。輸入一段人體姿勢序列（Pose sequence）以及第一幀起始圖像，模型需預測後續完整的動作影片

📊Dataset:
- 內容: 包含訓練影片（影像幀）與對應的人體骨架圖（Pose labels）
- 訓練與測試差異: 訓練影片長度固定為 16 幀，但測試影片則長達 630 幀

🛠️核心技術:
1. 資料維度重排: 配合「循序漸進」的預測邏輯
   - 在訓練與驗證初期，會將資料維度從 (B, seq, C, H, W) 重排為 (seq, B, C, H, W)，也就是將「時間序列（seq）」移到第一維度
   - 因為影片預測是 Frame-by-frame 的過程。將時間維度置頂，程式才能透過迴圈一幀一幀提取資訊，並將「當前生成的結果」動態傳給「下一幀」作為預測基礎

2. Teacher Forcing (TF): 從「輔助輪」到「獨立行走」
   - 模型在訓練時會決定使用 Ground Truth 或上一幀生成的影像作為輸入
   - 目的: 在模型尚未學會畫圖前，利用真實影像引導，避免初期誤差迅速擴大
   - 雖然使用真實影像能加速收斂，但若過度依賴，模型在測試（沒有真實影像可看）時會因不習慣預測長影片而崩潰。因此，我們會隨著 Epoch 增加而降低 TF 比例，強迫模型學習獨立預測的能力

3. 重參數化 (Reparameterization Trick): 讓隨機性變得可微 (可 Backpropagation)
   - 我們希望模型能有「想像力」，所以會從一個分佈中「隨機抽樣」出一個潛在變數 z。但問題來了：「隨機」這個動作是沒有路徑可以追蹤的。如果最後畫出來的圖很醜，模型沒辦法檢討「隨機抽樣」這個步驟該怎麼改，因為它就是隨機的，梯度到這裡就斷掉了 
   - 目的: 解決隨機採樣無法 Backpropagation 的問題
   - 白話解決方案: 把「隨機性」跟「模型參數」拆開。ex: 想像你要機器人丟球，原本是跟它說「隨機丟個力量」，現在改成:
     - 機器人決定「平均力量 μ」和「力道變化 σ」
     - 我們從外面丟給它一顆「隨機骰子 ϵ」（標準高斯雜訊）
     - 機器人的最終力道就是: z = μ + ϵ x σ
     - 現在球丟歪了，我們雖然不能檢討那顆「外部骰子 ϵ」，但我們可以檢討機器人設定的「平均值 μ」跟「變化量 σ」。這樣梯度就能順利流回網路，讓模型學會如何控制產出分佈的中心和範圍
   - 透過將隨機性轉移到雜訊 ϵ 上，使 z 變為均值 μ 與標準差的線性組合，梯度才能順利流動，讓模型學會如何優化潛在空間
  
4. KL Annealing: 平衡「畫得像」與「分佈準」
   - 模型的 Loss 有兩部分: Reconstruction Loss (MSE, 畫得像不像) 和 KL Divergence（KLD, 分佈規不規整)
   - KL Vanishing: 「KL」就像是一個嚴格的老師，要求所有的 z 都要整整齊齊地符合高斯分佈。如果訓練第一天老師就全力施壓（β = 1），模型會因為太害怕違規，乾脆放棄畫細節，只畫一個最保險、模糊的圖來應付老師
   - 在 Loss 計算中對 KL Divergence（KLD）加入動態權重 β
   - 白話解決方案: 引入權重 β，讓老師的壓力「由輕變重」
     - 初期(β ≈ 0): 老師不管你，讓你放飛自我隨便畫，只要「畫得像（MSE 越小）」就好。模型這時會學到很多影像細節
     - 後期(β → 1): 老師開始要求整潔，模型這時已經學會畫畫了，它只要微調一下位置，讓分佈變規整即可
  - 兩種策略的差別:
    - Monotonic (單調增加): 老師越來越嚴格(慢慢增加 β)，適合追求穩定的學習過程
    - Cyclical (週期變化): 老師嚴格一陣子、放鬆一陣子。給模型「喘息」機會重新探索潛在空間，能有效避免模型陷入局部優化，通常能得到更好的特徵分布


💡實驗發現:
- 長影片預測問題：訓練僅用 16 幀，但測試需預測 630 幀。若模型過度依賴 Teacher Forcing（TF），預測長影片時容易崩潰 。因此，我選擇調低 TF 比例，強迫模型在訓練中學會減少依賴 Ground Truth
- 收斂性分析:
  - 無 Annealing: Loss 曲線會非常混亂且難以收斂
  - Monotonic: 收斂最平滑，能穩定地利用潛在空間
- KL Divergence 的意義：其本質是計算 z 分佈與標準高斯分佈間的機率差異，確保生成的影像具有一定的多樣性
